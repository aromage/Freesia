prototype은 실행컨텍스트, this, scope chain과 함께 javascript의 근본적인 부분 중 하나이다. prototype을 이해하고 있어야 javascript 에서 객체들이 어떤 요소를 사용할 수 있는지, 어떤 방식으로 상속되는지에 대해 이해할 수 있다.

상속의 경우 하위 Object의 prototype이 상위 Object의 prototype을 참조한다.
이 때도 .prototype은 생략이 가능하다. (생략이 됨으로써 scope chain과 비슷한 작동 방식을 보인다) 이 때 중요한 점은 prototype을 상속받는 것이지 Object를 상속받는 것이 아니기 때문에 상위 Object에서 prototype에 선언되어있지 않은 property 혹은 method는 하위 Object나 instance에서 접근할 수 없다. 이런 속성 혹은 메소드를 static property, static method 라고 한다
